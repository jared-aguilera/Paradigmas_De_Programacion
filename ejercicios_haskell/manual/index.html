Problemario de Haskell

Manual de usuario interactivo para los ejercicios de Programación Funcional.

¿Qué es la aplicación?

Esta herramienta es una aplicación por terminal desarrollada completamente en Haskell. Su propósito es servir como un problemario interactivo que agrupa y ejecuta ejercicios prácticos de programación funcional, basados en el libro "Ejercicios de programación funcional con Haskell".

El proyecto está dividido modularmente en dos archivos principales:

  • Ejercicios.hs: Contiene toda la lógica matemática, algoritmos y estructuras de datos.
  • Main.hs: Gestiona la entrada/salida y el sistema de menús interactivos.

¿Cómo funciona?

La aplicación presenta un menú principal. Al seleccionar un capítulo, se despliega un submenú con los ejercicios correspondientes. Además el sistema cuenta con validaciones para garantizar que las entradas inválidas no rompan el programa.

Temario Detallado

A continuación se detallan los ejercicios implementados, divididos por capítulo, abarcando desde las bases funcionales hasta el desarrollo para juegos.

Capítulo 1: Introducción a la programación funcional

Conceptos matemáticos básicos y su traducción a funciones de Haskell.

  • 1.1 Factorial: Cálculo de factorial usando recursividad y condicionales.
  • 1.2 Combinaciones: Cálculo del número de combinaciones dadas n y k.
  • 1.3 Impar: Comprobación de si un número es impar.
  • 1.4 Cuadrado: Obtención del cuadrado de un número.
  • 1.5 Suma de Cuadrados: Sumar cuadrados de los elementos de una lista.
  • 1.6 Raíces: Cálculo de raíces de ecuaciones de segundo grado.
  • 1.7 Valor Absoluto: Función que devuelve el valor absoluto de un entero.
  • 1.8 Signo: Determinación del signo de un número entero (1, -1, o 0).
  • 1.9 Anterior Natural: Retorna el número anterior de un natural (mayor a 0).
  • 1.10 Potencia: Cálculo matemático manual de x^n.

Capítulo 2: Números y funciones

Ejercicios orientados al procesamiento de listas básicas y mapeos aritméticos.

  • 2.1 Siguiente: Devuelve el sucesor de un número.
  • 2.2 Doble: Multiplica por 2 un número dado.
  • 2.3 Mitad: Retorna la mitad en número decimal.
  • 2.4 Inverso: Retorna 1/x validando divisiones por cero.
  • 2.5 Potencia de 2: Función para calcular 2^n.
  • 2.6 Es Positivo: Validación de números positivos.
  • 2.7 Suma Lista: Sumatoria total de una lista de enteros.
  • 2.8 Producto Lista: Multiplicación consecutiva de una lista.
  • 2.9 Conjunción (AND): Retorna True solo si todos en la lista booleana son True.
  • 2.10 Disyunción (OR): Retorna True si al menos un elemento es True.

Capítulo 3: Estructuras de datos

Implementación manual de las funciones core de manipulación de listas en Haskell.

  • 3.1 Concatenación: Unión de dos listas independientes.
  • 3.2 Head: Obtener el primer elemento de una lista (con validaciones).
  • 3.3 Tail: Obtener el resto de la lista tras quitar la cabeza.
  • 3.4 Last: Encontrar y devolver el último elemento.
  • 3.5 Inversa: Voltear completamente una lista.
  • 3.6 Length: Conteo manual de longitud de una lista.
  • 3.7 Pertenencia: Comprobar si un elemento x pertenece a la lista.
  • 3.8 Todos Pares: Valida que todos los números dentro del arreglo sean divisibles entre 2.
  • 3.9 Fst (First): Obtención de la primera parte de una tupla.
  • 3.10 Snd (Second): Obtención de la segunda parte de una tupla.

Capítulo 4: Aplicaciones de programación funcional

Resolución de problemas clásicos y algoritmos complejos mediante el paradigma funcional.

  • 4.1 Segmentos iniciales: Generación de prefijos de una lista.
  • 4.2 Segmentos finales: Generación de sufijos de una lista.
  • 4.3 Segmentos: Obtención de todas las subcadenas continuas.
  • 4.4 Sublistas: Generación del conjunto potencia.
  • 4.5 Subconjunto: Comprobar si una lista está totalmente contenida en otra.
  • 4.6 Igualdad de Conjuntos: Comprobar que dos listas contienen los mismos valores.
  • 4.7 Permutaciones: Generar todas las posibles alteraciones de orden.
  • 4.8 Combinaciones: Seleccionar subgrupos de tamaño n.
  • 4.9 El problema de las Reinas: Resolución del clásico problema de ajedrez mediante backtracking.
  • 4.10 Números de Hamming: Generador de la secuencia infinita de Hamming.

Capítulo 5: Introducción a la programación funcional

Revisión de conceptos elementales bajo el enfoque de Claessen.

  • 5.1 Cuadrado: Implementación de la potencia al cuadrado elemental.
  • 5.2 Valor Absoluto: Re-evaluación del valor absoluto.

Capítulo 6: Modelización y tipos de datos

Abstracción matemática usando tipos funcionales.

  • 6.1 Concatenación de Listas: Unión elemental de secuencias.
  • 6.2 Inversa de Lista: Reversión eficiente de datos.

Capítulo 7: Recursión y tipos de datos

Enfoque profundo en llamadas iterativas mediante recursión estricta.

  • 7.1 Suma de Cuadrados Recursiva: Acumulación descendente.
  • 7.2 Potencia Recursiva: Evaluación de x^n multiplicando x a nivel inferior.

Capítulo 8: Listas y comprensión

Uso intensivo de generadores (list comprehensions) para lógica declarativa.

  • 8.1 Lista Ordenada: Evaluar que elementos subsiguientes conserven crecimiento <=.
  • 8.2 Posiciones (Buscador): Localizar todos los índices donde reside un elemento objetivo.

Capítulo 9: Funciones de entrada y salida

Manejo de Monadas IO para romper la pureza e imprimir en consola de forma controlada.

  • 9.1 Acción y Escritura: Ejecución forzada de secuencias putStrLn.
  • 9.2 Escritura de Tablas: Tabulación elegante de cadenas para listados o inventarios.

Capítulo 10: Tipos de datos recursivos

Estructuras jerárquicas como árboles para evaluar matemáticas.

  • 10.1 Expresiones Simples: Creación de tipo de dato Árbol soportando (Suma, Multiplicación, Número).
  • 10.2 Expresiones Generales: Expansión soportando (Suma, Resta, División, Producto).

Capítulo 11: Analizadores (Parsers)

Conversión y entendimiento de tokens de String a tipos fuertes.

  • 11.1 Uso de 'read': Conversiones directas nativas y sus fallos intrínsecos.
  • 11.2 Analizador Éxito: Creación del bloque base tipo ReadS que envuelve respuestas positivas puras.

Capítulo 12: Búsqueda en grafos y espacios de estados

Modelización de nodos y recorridos profundos.

  • 12.1 Grafo Dirigido: Definición semántica de vértices y sucesores estáticos.
  • 12.2 Pathfinder (Caminos): Búsqueda en profundidad (DFS) para trazar ruta entre origen y destino.

Capítulo 13: Juegos (EL GATO)

Aplicación del paradigma funcional para desarrollo de videojuegos en consola.

  • 13.1 Tres en Raya (El Gato): Implementación total de las mecánicas. Incluye control de turno mediante el algoritmo estricto de Minimax con Poda.
